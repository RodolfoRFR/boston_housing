Implementation: Define a Performance Metric¶

It is difficult to measure the quality of a given model without quantifying its performance over training and testing. This is typically done using some type of performance metric, whether it is through calculating some type of error, the goodness of fit, or some other useful measurement. For this project, you will be calculating the coefficient of determination, R2, to quantify your model's performance. The coefficient of determination for a model is a useful statistic in regression analysis, as it often describes how "good" that model is at making predictions.

The values for R2 range from 0 to 1, which captures the percentage of squared correlation between the predicted and actual values of the target variable. A model with an R2 of 0 always fails to predict the target variable, whereas a model with an R2 of 1 perfectly predicts the target variable. Any value between 0 and 1 indicates what percentage of the target variable, using this model, can be explained by the features. A model can be given a negative R2 as well, which indicates that the model is no better than one that naively predicts the mean of the target variable.

Answer:

By definition $R^2$ is equal to: $$ R = 1 -\frac{\sum_i {\left( y_i - f_i\right)}^2}{\sum_i {\left( \bar{y}_i - y_i\right)}^2}$$ where $y_i$ are the real tagert values, $\bar{y}_i$ their average and $f_i$ are the predicted target values. This metric tell us that if the sum of square of residuals, $\sum_i {\left( y_i - f_i\right)}^2$ (variation due to the prediction) is small compared to the total sum of squares, $\sum{\left( \bar{y}_i - y_i\right)}^2$ (variation within the dataset itself) the ratio is very small and $R^2$ will be close to 1. The predicted values followed the true values very closely with low variation. The score of 0.923 reflected this fact very well for this particular case.

Question 3 - Training and Testing¶

What is the benefit to splitting a dataset into some ratio of training and testing subsets for a learning algorithm?
Hint: What could go wrong with not having a way to test your model?

Answer:

When teaching your model how to predict the target value, the objective is to extract the general behavior embedded in the features of the dataset leaving the noise out. Since the data used for training will contain $useful \ information $ $+ \ ramdom\ deviations \ (noise)$, we want to avoid learning the dataset 'too well' (overfitting). Splitting your dataset, provided the features are initially uniformly distributed with respect to the target(s), minimizes overfitting problems.

The key concept is as follows. The training and testing set could be described as having correlated feature distribution but uncorrelated noise patterns. By splitting the dataset into two datasets, $\ training \ + \ training \ noise \ $ and $\ testing \ + \ testing \ noise \ $, we obtain independent sets with similar feature distributions but uncorrelated noise patterns. This facilitates the task of finding the correct set of estimator hyper-parameters (using Grid Search for example) by comparing what is learnt on the training set with the results of the testing set. Any leakage from the $training \ noise$ data present in the model can be found by finding a discrepancy between the training and testing performance metrics.

If instead we use the same data for both training and testing, the algorithm will potentially learn from the noise and we will not have a mechanism for comparison to determine the accuracy of the predictions.

Question 5 - Bias-Variance Tradeoff¶

When the model is trained with a maximum depth of 1, does the model suffer from high bias or from high variance? How about when the model is trained with a maximum depth of 10? What visual cues in the graph justify your conclusions?
Hint: How do you know when a model is suffering from high bias or high variance?

Answer:

When the model has a maximum depth of 1 we see a high bias (low training score) which implies the model is insensitive to additional information and will not learn from the dataset. For a max_depth of 10 there is an overfit and (high training score close to 1) the model is probably 'learning' from the noise as well as from the behavior we want to extract.

A high bias model has both low scores on training and validation (high error for both). When the model has high variance the training score is high (overfitting) and the validation score is low. In fact, a high training score is not by itself an indicator of overfitting, but the combination of both high training score (monotonically increasing) and low validation score is.

By not giving the model enough parameters (features) we obtain testing low scores. As we add more parameters, we are able to extract the generalization from the dataset, but if we continue increasing the complexity of the model overfitting begins and the test score starts to decrease, while the training score keeps improving.

What is the grid search technique and how it can be applied to optimize a learning algorithm?

Answer:

Every estimator has a set of hyper-parameters that are not changed through the learning iterations but heavily impact the model performance. The grid search technique iterates through either all parameter combinations (or a some subset of them following a particular rule). By sistematically searching the best set of parameters Grid Search can find the one that delivers the highest performance metric. Choosing between different types of Grid Search techniques invovles careful consideration of the computational resources available, time constraints and calculation objectives.

Question 8 - Cross-Validation¶

What is the k-fold cross-validation training technique? What benefit does this technique provide for grid search when optimizing a model?
Hint: Much like the reasoning behind having a testing set, what could go wrong with using grid search without a cross-validated set?

Answer:

Formally, the training procedure should include 3 subsets: train, validate and test data subsets. This is true when only one split is performed on the dataset. However, by performing K number of splits (obtaining K subsets), grouping K-1 as training and 1 subset for testing, iterating through each subset as the testing set calculating the performance metric for each training/testing selection, and averaging them to obtain a final metric value, we can eliminate the necessity for a validation subset. This method is called K-Fold.

The K-Fold technique works best when the sample distribution is random. If clusters of labeled data exist (bias), the choice of training and testing sets will influence the calculations. To reduce the effect of not having a perfectly random distribution across the whole dataset, cross-validation runs over all possible selection of test and training samples (for a given number of subsets. The number and size of the subsets determines the number of permutations and thus the calculation time.

    grid = GridSearchCV(regressor,params,scoring_fnc)

    # Fit the grid search object to the data to compute the optimal model
    grid = grid.fit(X, y)

    # Return the optimal model after fitting the data
    return grid.best_estimator_

Making Predictions¶

Once a model has been trained on a given set of data, it can now be used to make predictions on new sets of input data. In the case of a decision tree regressor, the model has learned what the best questions to ask about the input data are, and can respond with a prediction for the target variable. You can use these predictions to gain information about data where the value of the target variable is unknown such as data the model was not trained on.
